Detailed Description

Context holds details about location of flow that are hidden to concrete implementation. Context is also a container for services.

Date
Oct 4, 2013

Constructor & Destructor Documentation

Cross::Context::Context ( Context *  parentContext)
explicit

construct the context that will be a link to all services and flow. This will be passed/forked through various flows. A link to immediate parent must always be maintained. Never allow a NULL parent to be passed.

Parameters
parentContext- a link to parent context.
